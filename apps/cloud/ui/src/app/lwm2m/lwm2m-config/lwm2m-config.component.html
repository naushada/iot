Device Management Configuration

Devices register here after bootstrapping Registration lifetime pushed to devices at bootstrap Pushed in Server Object (OID 1) at bootstrap

Per-device DM credentials (PSK identity and key) are minted per-endpoint during provisioning and shown read-only on the Endpoints page — they are not configured here.

Device flow after bootstrap: Device sends Register to the DM URI above. Cloud DM server accepts the registration and the device appears as "online" in the endpoint list. Read / Write / Observe operations route through the DM server over the VPN tunnel.
